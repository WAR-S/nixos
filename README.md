# nixos-nettop

## Сборка установочного ISO

Образ — минимальный установщик NixOS с уже добавленными твоими пакетами (`git`, `vim`, `curl`, `htop`, `openssh`, `jq`, `yq`) и пользователем `wars` (sudo без пароля). Сеть в ISO не трогаем — на live-системе используется стандартная конфигурация установщика.

На хосте с Nix:

```bash
cd /path/to/nixos-nettop
nix --extra-experimental-features "nix-command flakes" build .#iso
```

Готовый образ будет в `result/iso/nixos-minimal-*.iso` (симлинк в store). Чтобы собрать **без симлинка** и скопировать ISO в текущую директорию (один файл):

```bash
nix --extra-experimental-features "nix-command flakes" run .#iso-build
```

После этого в каталоге появится файл `nixos-minimal-*.iso` и в выводе — его полный путь.

Запись на флешку:

```bash
# если использовали iso-build — образ в текущей директории
sudo dd if=nixos-minimal-*.iso of=/dev/sdX status=progress
# если собирали через nix build .#iso — образ в result/iso/
# sudo dd if=result/iso/nixos-minimal-*.iso of=/dev/sdX status=progress
sync
```

### Самоустановка на диск

Если загрузиться с флешки с **параметрами ядра** `nixos.autoInstall=1` и (опционально) `nixos.installDisk=/dev/sda`, то при старте автоматически выполнится:

1. разметка диска через disko (по `layers/os/disko-layout.nix`);
2. копирование конфига с ISO в `/etc/nixos` на целевом диске;
3. `nixos-install --flake ...#edge-node`;
4. перезагрузка.

Как включить: в меню загрузки (GRUB) добавь к строке ядра в конец: `nixos.autoInstall=1 nixos.installDisk=/dev/sda` (диск можно сменить). Без этих параметров загрузка как обычный live-образ, установка вручную.

### Уменьшение размера образа

- В конфиге ISO включено сжатие squashfs: `xz -Xdict-size 100%` — образ меньше, сборка дольше.
- Если нужна быстрая сборка, в `systems/iso/auto-install.nix` можно заменить на `isoImage.squashfsCompression = "xz";` или `"zstd"`.
- Лишние пакеты в `layers/os/packages.nix` для live-образа можно вынести в отдельный список и не добавлять их в ISO.

# Техническое задание

## Миграция инфраструктуры на NixOS с использованием слойной архитектуры

---

# 1. Цель проекта

Разработать воспроизводимую инфраструктуру на базе NixOS, обеспечивающую:

* декларативное описание инфраструктуры
* воспроизводимость конфигураций
* автоматическую сборку образов
* автоматическую установку системы
* управление контейнерной платформой
* GitOps-деплой приложений

Инфраструктура должна реализовываться с использованием **слойной архитектурной модели**.

---

# 2. Архитектурная модель

Система должна быть реализована в виде **наборa независимых слоев (layers)**.

Каждый слой:

* имеет собственную область ответственности
* реализуется как отдельный модуль конфигурации
* может переиспользоваться
* может быть отключен или расширен

Слои применяются последовательно.

## Общая схема слоев

```
Hardware Layer
      ↓
OS Layer
      ↓
Network Layer
      ↓
Security Layer
      ↓
Container Platform Layer
      ↓
Application Layer
```

---

# 3. Hardware Layer

## Назначение

Слой отвечает за взаимодействие операционной системы с аппаратным обеспечением.

## Функциональные требования

Должны быть реализованы:

* конфигурация устройств
* правила автоматического обнаружения устройств
* управление сетевыми интерфейсами
* поддержка модемов и сетевых устройств

## Компоненты

* правила устройств через udev
* поддержка:

  * USB modem
  * Ethernet
  * WiFi адаптеров

## Функции

* автоматическое определение типа сетевого подключения
* автоматическая активация соответствующих сервисов

---

# 4. OS Layer

## Назначение

Слой отвечает за базовую конфигурацию операционной системы.

## Функциональные требования

Слой должен обеспечивать:

* настройку базовой системы
* конфигурацию systemd сервисов
* управление пакетами
* управление пользователями
* конфигурацию bootloader

## Компоненты

* системные пакеты
* базовые сервисы
* управление пользователями
* файловые системы
* системные параметры ядра
